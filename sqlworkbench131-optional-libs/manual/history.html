3. Change log

Changes from build 130 to build 131

Enhancements

  • When using a file as the input for the DataPumper, it's now possible to change the option for the WbImport in the DataPumper dialog.
  • It's now possible to hide the display of the statements in the profile's connect script.
  • For DB2 for i the JDBC ClientInfo is now initialized.
  • It's now possible to generate a SQL condition from a filter definition
  • When importing spreadsheets it's now possible to provide a sheet name to table name mapping, if the sheet names don't reflect the target tables
  • In the DbTree and DbExplorer it's now possible to set the default sort order for table columns by name rather than by position
  • It's now possible to specify the format of column defined as TIMESTAMP WITH TIME ZONE separately

Bug fixes

  • Copy to clipboard as JSON did not work
  • WbSchemaDiff did not evaluate the ON UPDATE or ON DELETE options of foreign keys.
  • In the DbTree it was no longer possible to jump to entries using the keyboard.
  • The output of commands run through WbInclude in the connect script weren't displayed.
  • Profile variables were not used in the profile's connect script.
  • With certain Look & Feels, the context menu on an editor or result tab was displayed for the currently selected tab, not the tab on which the context menu was invoked.
  • When reloading a result that was sorted, the column widths would not always be restored correctly.
  • Better handle errors when loading profiles during startup
  • When copying data as DbUnit XML, timestamp values were not formatted properly.
  • Changing a table's definition in the DbExplorer would not generate the corresponding DDL any more.
  • When using real tabs in the editor, the position of the following characters still wasn't calculated correctly
  • For PostgreSQL, TYPEs where no longer retrieved in the DbExplorer or DbTree
  • In the DbTree the list of table columns could no longer be sorted by name
  • TIMESTAMP or TIMESTAMP WITH TIME ZONE columns were not imported correctly from XML files
  • It was no longer possible to quit SQL Workbench without saving modified files

The full release history is available at the SQL Workbench/J homepage
